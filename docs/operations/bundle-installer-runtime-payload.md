# Bundle Installer Runtime Payload Guardrail

[Back to Main README](../../README.md) | [Back to Docs Hub](../README.md)

This note captures a recurring Windows bundle-installer failure where installed tray services do not boot after setup.

## Symptom pattern

- Installer completes successfully.
- Service/web tray shortcuts exist.
- Running tray host appears to do nothing, or exits immediately.
- Installed runtime under Program Files is missing expected Electron files (for example locales and `.pak` files).

## Root cause

In the NSIS script generated by `scripts/build-bundle-installer.mjs`, runtime copy logic previously short-circuited when `runtime\\electron\\electron.exe` already existed.

That made partial or stale runtime installs survive repair/upgrades.

## Permanent fix (applied)

- File: `scripts/build-bundle-installer.mjs`
- Function: `EnsureRuntimePayload`
- Change: removed the `IfFileExists ... runtime_done` guard, so runtime payload is always recopied on install.

## Release-time verification checklist

1. Build installer with:
   - `npm run dist:bundle-installer:win`
2. Install bundle on a machine with a previous install present (upgrade/repair scenario).
3. Verify installed runtime folder contains full Electron payload under:
   - `C:\\Program Files\\Local Game Gallery\\runtime\\electron`
4. Verify service starts:
   - Start menu shortcut: `Gallery Service Tray`
   - or direct command:
     - `"C:\\Program Files\\Local Game Gallery\\runtime\\electron\\electron.exe" "C:\\Program Files\\Local Game Gallery\\dist-electron\\electron\\service-tray.js"`
5. Confirm service endpoint responds:
   - `http://127.0.0.1:37995/api/health`

## Fast diagnostic commands

PowerShell checks for missing runtime files:

```powershell
$runtime = "$env:ProgramFiles\Local Game Gallery\runtime\electron"
Get-ChildItem $runtime | Select-Object Name
Get-ChildItem "$runtime\resources" | Select-Object Name
Test-Path "$runtime\locales"
```

If runtime looks incomplete after install, treat installer artifact as invalid and rebuild/reinstall.
